API Reference

Formulas

Gamen.Formula — Type
Formula

Abstract base type for all modal logic formulas (Definition 1.2, B&D).

Gamen.Bottom — Type
Bottom <: Formula

The propositional constant for falsity ⊥ (Definition 1.1, item 1).

Gamen.Atom — Type
Atom <: Formula

A propositional variable identified by a symbol (Definition 1.1, item 2). Supports both named atoms like Atom(:p) and indexed atoms like Atom(0).

Gamen.Not — Type
Not <: Formula

Negation of a formula (Definition 1.2, item 3).

Gamen.And — Type
And <: Formula

Conjunction of two formulas (Definition 1.2, item 4).

Gamen.Or — Type
Or <: Formula

Disjunction of two formulas (Definition 1.2, item 5).

Gamen.Implies — Type
Implies <: Formula

Material conditional (Definition 1.2, item 6).

Gamen.Iff — Type
Iff <: Formula

Biconditional. A ↔ B abbreviates (A → B) ∧ (B → A) (Definition 1.3, item 2). Represented as its own type for convenience.

Gamen.Box — Type
Box <: Formula

The necessity modal operator □ (Definition 1.2, item 7).

Gamen.Diamond — Type
Diamond <: Formula

The possibility modal operator ◇ (Definition 1.2, item 8).

□(operand::Formula)

Unicode alias for Box. Type \square<tab> in the Julia REPL.

Gamen.◇ — Type
◇(operand::Formula)

Unicode alias for Diamond. Type \diamond<tab> in the Julia REPL.

Gamen.Top — Function
Top()

⊤ abbreviates ¬⊥ (Definition 1.3, item 1).

Gamen.is_modal_free — Function
is_modal_free(f::Formula) -> Bool

Return true if the formula contains no □ or ◇ operators.

Gamen.atoms — Function
atoms(f::Formula) -> Set{Symbol}

Collect all propositional variable names appearing in a formula.

Kripke Structures

Gamen.KripkeFrame — Type
KripkeFrame

A Kripke frame ⟨W, R⟩ consisting of a nonempty set of worlds W and a binary accessibility relation R on W (Definition 1.6, B&D).

Gamen.KripkeModel — Type
KripkeModel

A model M = ⟨W, R, V⟩ where V is a valuation function assigning to each propositional variable p a set V(p) of worlds where p is true (Definition 1.6, B&D).

The valuation maps atom names (symbols) to sets of worlds, matching the book's convention: V(p) ⊆ W.

Gamen.accessible — Function
accessible(frame::KripkeFrame, world::Symbol)

Return the set of worlds accessible from world. When Rww' holds, we say w' is accessible from w (Definition 1.6, B&D).

Semantics

Gamen.satisfies — Function
satisfies(model::KripkeModel, world::Symbol, formula::Formula) -> Bool

Determine whether formula is true at world in model, written M, w ⊩ A in the book (Definition 1.7, B&D).

Gamen.is_true_in — Function
is_true_in(model::KripkeModel, formula::Formula) -> Bool

A formula A is true in a model M, written M ⊩ A, if and only if M, w ⊩ A for every w ∈ W (Definition 1.9, B&D).

Gamen.is_valid — Function
is_valid(formula::Formula, models) -> Bool

A formula A is valid in a class of models C if it is true in every model in C (Definition 1.11, B&D). Pass any iterable collection of models.

Gamen.entails — Function
entails(model::KripkeModel, premises, conclusion::Formula) -> Bool

A set of formulas Γ entails A in model M if for every world w ∈ W, if M, w ⊩ B for every B ∈ Γ, then M, w ⊩ A (Definition 1.23, B&D).

If premises is a single formula, it is treated as a singleton set.

Frame Properties

Gamen.is_reflexive — Function
is_reflexive(frame::KripkeFrame) -> Bool

A frame is reflexive if every world accesses itself: ∀w ∈ W, Rww (Definition 2.3, B&D).

Gamen.is_symmetric — Function
is_symmetric(frame::KripkeFrame) -> Bool

A frame is symmetric if accessibility is symmetric: ∀w, w' ∈ W, Rww' → Rw'w (Definition 2.3, B&D).

Gamen.is_transitive — Function
is_transitive(frame::KripkeFrame) -> Bool

A frame is transitive if accessibility is transitive: ∀w, w', w'' ∈ W, (Rww' ∧ Rw'w'') → Rww'' (Definition 2.3, B&D).

Gamen.is_serial — Function
is_serial(frame::KripkeFrame) -> Bool

A frame is serial if every world has at least one successor: ∀w ∈ W, ∃w' ∈ W, Rww' (Definition 2.3, B&D).

Gamen.is_euclidean — Function
is_euclidean(frame::KripkeFrame) -> Bool

A frame is euclidean if: ∀w, w', w'' ∈ W, (Rww' ∧ Rww'') → Rw'w'' (Definition 2.3, B&D).

Gamen.is_partially_functional — Function
is_partially_functional(frame::KripkeFrame) -> Bool

A frame is partially functional if every world has at most one successor: ∀w∀u∀v((Rwu ∧ Rwv) → u = v) (Table frd.2, B&D).

Gamen.is_functional — Function
is_functional(frame::KripkeFrame) -> Bool

A frame is functional if every world has exactly one successor: ∀w∃v∀u(Rwu ↔ u = v) (Table frd.2, B&D).

Equivalently, a frame is functional iff it is both serial and partially functional.

Gamen.is_weakly_dense — Function
is_weakly_dense(frame::KripkeFrame) -> Bool

A frame is weakly dense if every step can be decomposed into two steps: ∀u∀v(Ruv → ∃w(Ruw ∧ Rwv)) (Table frd.2, B&D).

Gamen.is_weakly_connected — Function
is_weakly_connected(frame::KripkeFrame) -> Bool

A frame is weakly connected if any two successors of a world are related or identical: ∀w∀u∀v((Rwu ∧ Rwv) → (Ruv ∨ u = v ∨ Rvu)) (Table frd.2, B&D).

Gamen.is_weakly_directed — Function
is_weakly_directed(frame::KripkeFrame) -> Bool

A frame is weakly directed (has the "diamond property" or "confluence") if any two successors of a world have a common successor: ∀w∀u∀v((Rwu ∧ Rwv) → ∃t(Rut ∧ Rvt)) (Table frd.2, B&D).

Gamen.is_equivalence_relation — Function
is_equivalence_relation(frame::KripkeFrame) -> Bool

A frame's accessibility relation is an equivalence relation iff it is reflexive, symmetric, and transitive (Definition frd.11, B&D).

Gamen.is_universal — Function
is_universal(frame::KripkeFrame) -> Bool

A frame's accessibility relation is universal if every world is accessible from every world: ∀u,v ∈ W, Ruv (Definition frd.11, B&D).

Gamen.is_valid_on_frame — Function
is_valid_on_frame(frame::KripkeFrame, formula::Formula) -> Bool

A formula A is valid on a frame F = ⟨W, R⟩ if it is true in every model M = ⟨W, R, V⟩ based on F (Definition 2.1, B&D).

This enumerates all possible valuations for the propositional variables in the formula, so it is only tractable for small frames and formulas.

Gamen.standard_translation — Function
standard_translation(φ::Formula, [x::FOVar]) -> FOFormula

Compute the standard translation STₓ(φ) of a modal formula φ into first-order logic (Definition frd.15, B&D).

The translation maps:

  • Propositional variable p to unary predicate P_p(x)
  • □ψ to ∀y (Q(x, y) → STᵧ(ψ))
  • ◇ψ to ∃y (Q(x, y) ∧ STᵧ(ψ))

The binary predicate Q represents the accessibility relation.

Example

julia> using Gamen

julia> p = Atom(:p);

julia> standard_translation(□(Implies(p, p)))
∀y₁ (Q(x, y₁) → (P_p(y₁) → P_p(y₁)))

julia> standard_translation(◇(p))
∃y₁ (Q(x, y₁) ∧ P_p(y₁))

Axiomatic Derivations

Substitution and Tautologies

Gamen.substitute — Function
substitute(φ::Formula, σ) -> Formula

Apply substitution σ to formula φ, replacing each Atom(name) whose name is a key in σ with the corresponding formula. σ should be a Dict{Symbol, <:Formula} or similar mapping.

Gamen.is_tautology — Function
is_tautology(φ::Formula) -> Bool

Check whether the modal-free formula φ is a propositional tautology, i.e., true under every truth-value assignment. Throws ArgumentError if φ contains modal operators.

Gamen.is_tautological_instance — Function
is_tautological_instance(φ::Formula) -> Bool

Check whether φ is an instance of a propositional tautology (Definition 3.3, item 1, B&D). Extracts the propositional skeleton by treating atoms and modal subformulas as propositional variables, then checks whether the skeleton is a tautology.

Modal Systems

Gamen.ModalSystem — Type
ModalSystem

A modal proof system defined by a name and a set of axiom schemas (Definition 3.9, B&D). Every normal modal system includes the K and Dual axiom schemas; additional schemas determine the specific system.

Gamen.SYSTEM_K — Constant

The basic modal system K with axioms K and Dual.

Gamen.SYSTEM_KT — Constant

KT = K + Schema T (□A → A).

Gamen.SYSTEM_KD — Constant

KD = K + Schema D (□A → ◇A).

Gamen.SYSTEM_KB — Constant

KB = K + Schema B (A → □◇A).

Gamen.SYSTEM_K4 — Constant

K4 = K + Schema 4 (□A → □□A).

Gamen.SYSTEM_K5 — Constant

K5 = K + Schema 5 (◇A → □◇A).

Gamen.SYSTEM_S4 — Constant

S4 = K + T + 4.

Gamen.SYSTEM_S5 — Constant

S5 = K + T + 5.

Derivations

Gamen.Justification — Type
Justification

Abstract type for proof step justifications in a derivation.

Gamen.Tautology — Type
Tautology <: Justification

The formula is a tautological instance (Definition 3.3, item 1, B&D).

Gamen.AxiomInst — Type
AxiomInst <: Justification

The formula is an instance of an axiom schema (Definition 3.3, item 2, B&D).

Gamen.ModusPonens — Type
ModusPonens <: Justification

The formula follows by modus ponens from two earlier steps (Definition 3.1, B&D). minor is the step index of A; major is the step index of A → B.

Gamen.Necessitation — Type
Necessitation <: Justification

The formula □A follows by necessitation from an earlier step proving A (Definition 3.2, B&D).

Gamen.ProofStep — Type
ProofStep

A single step in a derivation: a formula together with its justification.

Gamen.Derivation — Type
Derivation

A derivation (Definition 3.3, B&D): a sequence of proof steps where each step is justified as a tautological instance, axiom instance, or follows by modus ponens or necessitation from earlier steps.

Gamen.conclusion — Function
conclusion(d::Derivation) -> Formula

Return the formula proved by the derivation (its last step).

Gamen.is_valid_derivation — Function
is_valid_derivation(system::ModalSystem, deriv::Derivation) -> Bool

Check whether every step in deriv is correctly justified in the given modal system (Definition 3.10, B&D).

Dual Formulas

Gamen.dual — Function
dual(φ::Formula) -> Formula

Compute the dual of formula φ (Definition 3.26, B&D). The dual swaps ⊥ ↔ ⊤, ∧ ↔ ∨, and □ ↔ ◇. Atoms are negated, and ¬ distributes through the dual. The key property is that ⊢ A ↔ ¬(dual(A)).
